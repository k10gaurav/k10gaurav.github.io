WSO2 Identity Server Developer Exam v5

Authentication & Authorization
1. Which protocol is primarily used for API security in WSO2 Identity Server?
A. OAuth 2.0
B. SAML 1.1
C. LDAP
D. WS-Trust
OAuth 2.0 is the standard protocol used for API security in WSO2 Identity Server, providing token-based authorization for APIs.
2. What is the purpose of the 'scope' parameter in OAuth 2.0?
A. To specify token expiration time
B. To limit application access to specific resources
C. To define encryption algorithm
D. To identify the user
The scope parameter defines the specific permissions being requested by the client application, limiting access to certain resources or actions.
3. Which component in WSO2 IS is responsible for validating access tokens?
A. Key Manager
B. User Store
C. Identity Provider
D. Entitlement Server
The Key Manager component in WSO2 IS handles OAuth operations including token validation, issuance, and management.
Identity Federation
4. What is the primary purpose of SAML 2.0 in WSO2 IS?
A. User provisioning
B. Web-based single sign-on
C. Password policy enforcement
D. API security
SAML 2.0 is primarily used for web-based single sign-on (SSO) between different security domains or organizations.
5. What must be exchanged between Identity Provider and Service Provider before configuring SAML SSO?
A. User credentials
B. Metadata files
C. SSL certificates
D. Access tokens
SAML metadata files containing configuration details must be exchanged between IdP and SP before setting up SSO.
User Management
6. Which protocol is used for user provisioning in WSO2 IS?
A. OAuth 2.0
B. SCIM
C. XACML
D. SAML
SCIM (System for Cross-domain Identity Management) is the standard protocol used for user provisioning operations.
7. Where would you configure password policies in WSO2 IS?
A. Identity → Claims → List
B. Identity → Service Providers
C. Identity → Identity Providers
D. Identity → Roles
Password policies are configured under Identity → Claims → List by modifying password-related claims in the http://wso2.org/claims dialect.
Adaptive Authentication
8. What scripting language is used for adaptive authentication scripts in WSO2 IS?
A. JavaScript
B. Python
C. Groovy
D. Ruby
WSO2 IS uses JavaScript for writing adaptive authentication scripts that execute during the authentication flow.
9. Which of these is NOT a standard authentication step option in adaptive auth?
A. Basic
B. FIDO
C. Kerberos
D. SMS OTP
Kerberos is not available as a standard authentication step option in WSO2 IS adaptive authentication configurations.
API Security
10. What is the purpose of the 'aud' claim in a JWT access token?
A. Identifies the token issuer
B. Specifies the intended recipient
C. Contains user attributes
D. Defines token expiration
The 'aud' (audience) claim identifies the intended recipients of the token - the services that should accept the token.
11. Which grant type is most secure for server-side web applications?
A. Authorization Code
B. Implicit
C. Password
D. Client Credentials
The Authorization Code grant is most secure for server-side apps as tokens are never exposed to the user agent/browser.
Configuration & Deployment
12. Which file is the primary configuration file in WSO2 IS v5?
A. deployment.toml
B. carbon.xml
C. axis2.xml
D. user-mgt.xml
deployment.toml is the main configuration file in WSO2 IS v5, replacing multiple XML files from previous versions.
13. Where are custom authenticators deployed in WSO2 IS?
A. repository/components/lib
B. repository/components/dropins
C. repository/components/plugins
D. repository/components/extensions
Custom authenticators should be deployed as OSGi bundles in the repository/components/dropins directory.
Troubleshooting
14. Which log file contains authentication-related events?
A. wso2carbon.log
B. audit.log
C. http_access.log
D. trace.log
audit.log contains authentication and authorization related events, useful for troubleshooting login issues.
15. What command would you use to check the status of WSO2 IS?
A. wso2server.sh status
B. status.sh
C. carbon.sh --status
D. server-status.sh
'wso2server.sh status' command shows whether the server is running and its process ID.
